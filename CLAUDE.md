# Локальные правила проекта AgroNDVI

## Цель проекта
Пет-проект по ML/геоданным для портфолио. Домен -- агро / спутниковый мониторинг полей. Цель трудоустройства -- ML/DS-команды в банковском агро-сегменте (РСХБ-ЦР, Сбер для агробизнеса, Совкомбанк), агрохолдинги (ЭФКО, Русагро, Содружество), агростраховые (Совкомбанк Страхование, Ингосстрах).

Это второй пет-проект Валентюка Евгения. Первый -- MiniProctor (CV-прокторинг на MediaPipe + YOLOv8) -- закрыт и залит на GitHub. AgroNDVI расширяет портфолио в сторону геоданных и time-series.

## Порядок выполнения задач
1. Перед началом работы -- `TodoWrite`, разбить запрос на атомарные задачи.
2. Отмечать выполненное сразу, не батчить.
3. По завершении сессии -- зафиксировать результат в `AgroNDVI-results.md`. Формат: дата и время в `## YYYY-MM-DD HH:MM` (через `date "+%Y-%m-%d %H:%M"`, не придумывается), что сделано, какие файлы созданы/изменены, какие метрики получены.

Исключения: короткие ответы на вопросы без изменения файлов -- todo и лог не нужны.

## Лог сессий
- Технический лог: `AgroNDVI-results.md` в корне проекта. Заголовок записи -- системные дата и время в формате `## YYYY-MM-DD HH:MM`.
- Параллельный «человеческий» лог: `AgroNDVI-journal.md` в корне. Простыми словами, от первого лица, по дням, без технических терминов. Формат подходит для постов и рекрутёров.
- После каждого дня обновлять оба файла. В журнал пишем 5-7 пунктов «Я сделал X» простым языком и одну строку «Итог дня».
- Подробные трассы экспериментов -- в `docs/experiments/<дата>.md`.

## Стек
- Python 3.11+, venv в `.venv/`.
- **Геоданные:** `rasterio`, `geopandas`, `shapely`, `pyproj`, `folium`.
- **ML:** `lightgbm`, `scikit-learn`, опционально `prophet` или `darts` для time-series.
- **Numerics:** `numpy`, `pandas`.
- **UI:** `streamlit`, `plotly`.
- **Источники данных (бесплатные):**
    - **Sentinel-2** через AWS S3 (`sentinel-s2-l2a` bucket, requester pays) или Copernicus Browser;
    - **Open Meteo** -- историческая погода, бесплатно без ключа;
    - **NASA POWER** -- солнечная радиация, температура, осадки;
    - **OpenStreetMap** -- границы полей и подложки карт;
    - **Росстат** -- исторические урожайности по регионам.
- **Без LLM** -- задача математико-вычислительная, нейросетевые компоненты не нужны до тех пор, пока не решим добавить U-Net для сегментации полей.

## Структура
- `data/` -- скачанные Sentinel-tile, shapefile с полями, погода, урожайности. **Не коммитим в git** (могут быть гигабайты).
- `src/` -- исходный код пайплайна.
- `notebooks/` -- Jupyter-ноутбуки для exploratory analysis.
- `docs/` -- brief, architecture, отчёты по экспериментам, metrics, portfolio-report для нанимателя.
- `models/` -- сохранённые LightGBM-модели (`.pkl` или `.lgb`).
- `.claude/skills/` -- локальные скиллы (по необходимости).

## Документация
- `README.md` -- краткая сводка для GitHub.
- `docs/brief.md` -- полная постановка задачи, план, риски, open questions.

- `docs/experiments/` -- отчёты по экспериментам с метриками.
